# brew-snapshot Design Spec

**Date:** 2026-04-10
**Status:** Approved

## Goal

A Homebrew tap (`AndrewDongminYoo/tap`) that provides a CLI tool for snapshotting and restoring a macOS Homebrew environment. Distributed via `brew install`, configured via `brew-snapshot setup`.

## Repository Structure

**Tool source** lives at `~/Development/01_personal/brew-snapshot` and is published as GitHub repo `AndrewDongminYoo/homebrew-tap`.

**State directory** is `~/.local/share/brew-snapshot/` — a plain data folder, not a git repo. Path can be overridden with `$BREW_SNAPSHOT_DIR`.

```plaintext
brew-snapshot/
  bin/
    brew-snapshot                    # entry point — subcommand dispatch
  libexec/
    commands/
      snapshot.sh
      restore.sh
      status.sh
      setup.sh
  share/
    brew-snapshot.plist.template     # launchd LaunchAgent template
  Formula/
    brew-snapshot.rb
  README.md
  CLAUDE.md
```

## State Directory Layout

| File                | Generated by                      | Purpose                               |
| ------------------- | --------------------------------- | ------------------------------------- |
| `Brewfile`          | `brew bundle dump`                | Declarative reinstall manifest        |
| `Brewfile.lock`     | `brew info --json=v2 --installed` | Version history JSON                  |
| `Brewfile.deps`     | `brew deps --tree --installed`    | Dependency tree                       |
| `Brewfile.taps`     | `brew tap`                        | Active taps list                      |
| `Brewfile.refs`     | tap git rev-parse loop            | Tap commit hashes for reproducibility |
| `last_snapshot_utc` | `date -u`                         | Timestamp of last snapshot            |

`Brewfile` is the source of truth for reinstallation. `Brewfile.*` files are all regeneratable snapshots. None are version-controlled.

## CLI Interface

```plaintext
brew-snapshot snapshot [--greedy]   # snapshot current state
brew-snapshot restore               # restore from Brewfile on new Mac
brew-snapshot status                # show last snapshot time + package counts
brew-snapshot setup                 # install launchd LaunchAgent
brew-snapshot help                  # usage
```

### `snapshot` execution order

1. `brew update`
2. `brew upgrade` (default) / `brew upgrade --greedy` (with `--greedy` flag)
3. `brew bundle dump --force` → `Brewfile`
4. `brew info --json=v2 --installed` → `Brewfile.lock`
5. `brew deps --tree --installed` → `Brewfile.deps`
6. `brew tap` → `Brewfile.taps`
7. tap git rev-parse loop → `Brewfile.refs`
8. `date -u` → `last_snapshot_utc`

Script uses `set -euo pipefail`. Steps 3–8 run only after upgrade completes.

### `restore` execution

1. `brew bundle --file="$STATE_DIR/Brewfile"`
2. Print note: check `Brewfile.lock` for version differences on critical packages

### `setup` execution

1. Read `share/brew-snapshot.plist.template`
2. Substitute `__USER__` and `__STATE_DIR__` and `__BIN__` with actual values
3. Write to `~/Library/LaunchAgents/com.$USER.brew-snapshot.plist`
4. `launchctl unload` (ignore error) then `launchctl load`

## Homebrew Formula

```ruby
class BrewSnapshot < Formula
  desc "Snapshot and restore your Homebrew environment"
  homepage "https://github.com/AndrewDongminYoo/homebrew-tap"
  url "..."
  version "0.1.0"

  def install
    bin.install "bin/brew-snapshot"
    libexec.install Dir["libexec/*"]
    (share/"brew-snapshot").install "share/brew-snapshot.plist.template"
  end

  def caveats
    <<~EOS
      Run setup to enable automatic snapshots on login:
        brew-snapshot setup

      State directory: ~/.local/share/brew-snapshot/
      Override with: export BREW_SNAPSHOT_DIR=/your/path
    EOS
  end
end
```

## Installation Flow

```bash
brew tap AndrewDongminYoo/tap
brew install brew-snapshot
brew-snapshot setup       # register launchd agent
brew-snapshot snapshot    # first snapshot
```

## What Does NOT Belong Here

- Full version pinning for all packages (conflicts with Homebrew's model)
- `brew pin` on all formulae (causes upgrade conflicts)
- A guarantee of exact version reproduction (Homebrew does not support this)
- Cross-platform support (macOS + Homebrew only)
